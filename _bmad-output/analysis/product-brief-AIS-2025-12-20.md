# Product Brief: AIS

**Date:** 2025-12-20
**Author:** Adityajain

---

## Executive Summary

**AIS (Aadinath Information System)** is a comprehensive ERP system purpose-built for rolling mill operations under the AA (Double A) brand. It transforms a business currently operating on monthly invoice reconciliation and gut-feel decisions into a data-driven operation with real-time visibility, intelligent planning, and motivated teams.

The system uniquely combines traditional ERP capabilities (procurement, sales, attendance, reporting) with Computer Vision modules that capture objective production data — eliminating the ambiguity that plagues manual reporting. Integrated weighbridge and vehicle detection provide unchallengeable proof of material movement, while tiered worker presence tracking builds accountability across the factory floor. Built on Toyota Production System principles, AIS delivers visual management, waste elimination, and continuous improvement as native capabilities, not afterthoughts.

**Core Promise:** Instantaneous clarity on business performance, enabling confident decisions, strategic growth planning, and a motivated workforce that sees how their work contributes to company success.

---

## Core Vision

### Problem Statement

Rolling mill owners and production managers operate in a fog of ambiguity. Critical business questions — *How much did we actually produce? Why was output low? Are we profitable on this order? What should we manufacture next?* — cannot be answered with confidence. Data exists only in monthly invoice reconciliation, missing the operational reality that drives results.

### Problem Impact

| Impact Area | Consequence |
|-------------|-------------|
| **Decision Quality** | Sale prices driven by emotion and competition guessing, not actual cost knowledge |
| **Production Planning** | Next product chosen by "what looks short" — no calculation, pure estimation |
| **Problem Detection** | Issues discovered weeks later during monthly reconciliation — too late to act |
| **Growth Planning** | Impossible without knowing daily/weekly/monthly metrics and trends |
| **Accountability** | No objective data means disputes, blame games, and eroded trust |
| **Historical Learning** | No trends to analyze — same mistakes repeated, improvements untracked |
| **Weighbridge Security** | CCTV unwatched, weights manually entered — owner skepticism about material accuracy persists |
| **Vehicle Tracking** | No automatic vehicle log or gate pass — material movement unverifiable |
| **Worker Presence** | Binary attendance only — no visibility into actual time at station or hourly accountability |

### Why Existing Solutions Fall Short

| Current Approach | Gap |
|------------------|-----|
| **Monthly invoice/bill reconciliation** | Too late, too incomplete — misses operational data entirely |
| **Paper registers & manual logs** | Prone to human error, disputes over accuracy, no historical analysis |
| **WhatsApp updates & verbal reports** | Unstructured, unverifiable, lost in chat history |
| **Gut feel & floor walks** | Doesn't scale, depends on owner presence, no data trail |
| **Generic ERP systems** | Not built for rolling mill realities — miss production nuances, no CV integration |
| **Unwatched CCTV** | Cameras exist but no continuous monitoring — security gaps persist |

### Proposed Solution

AIS delivers **real-time operational visibility** through four integrated pillars:

**Pillar 1 — Digital ERP Foundation:**
- Mobile-first data capture at source (procurement, sales, attendance, downtime)
- Structured records replacing paper registers and WhatsApp chaos
- Automated calculations eliminating manual errors
- Role-based dashboards showing each person their key metrics

**Pillar 2 — Computer Vision Intelligence:**
- Objective production time tracking (running vs. breaks vs. downtime)
- Answers the unanswerable: "What actually happened this shift?"
- Hourly worker presence detection across factory cameras (v2)
- Validates and enriches ERP data with ground truth

**Pillar 3 — Weighbridge & Security Integration:**
- Vehicle detection with automatic number plate recognition (add camera if needed)
- Direct weighbridge integration — gross, tare, net weight auto-captured
- Vehicle log with timestamp, plate number, weights, material type, purpose
- Auto-generated gate pass with complete audit trail
- Eliminates owner skepticism through unchallengeable proof

**Pillar 4 — Analysis & Intelligence:**
- Historical trends across days, weeks, months, years
- Analysis Playground for free-form correlation discovery
- External intelligence integration (competition rates, market trends)
- Growth obstacle identification and strategic planning support

### Key Differentiators

| Differentiator | Strategic Value |
|----------------|-----------------|
| **CV + ERP Integration** | No rolling mill ERP offers this — objective production data without trusting manual logs |
| **Weighbridge + Vehicle Detection** | Auto-captured weights + number plates = unchallengeable material movement proof |
| **Tiered Worker Presence** | From daily attendance (v1) to hourly station tracking (v2) — scalable accountability |
| **Role-Based Visibility** | Privacy-respecting dashboards — Foreman sees breakdowns, Labour supervisor sees production time vs breaks, Owner sees everything |
| **TPS Principles Native** | Visual management, waste elimination, Kaizen loops built into the system DNA |
| **Analysis Playground** | Discover correlations freely before formalizing — supports continuous improvement |
| **External Intelligence Layer** | Competition rates + market trends integrated for pricing and strategy |
| **Mobile-First Design** | Workers capture data on phones they already own — minimal infrastructure |

### Why Now

Two forces have converged to make AIS possible and necessary:

1. **Technology Accessibility** — Computer Vision, cloud infrastructure, and mobile apps have reached price points and maturity levels that make sophisticated systems achievable for mid-sized operations

2. **Personal Drive to Scale** — The ambition to professionalize operations, eliminate gut-feel management, and build a data-driven foundation for strategic growth

**Pillar 3 — Weighbridge & Security (Manual Integration)**

| Feature | MVP-Core Scope |
|---------|----------------|
| **Weighbridge Entry** | Manual entry with Hydra comparison validation |
| **Vehicle Log** | Timestamp, weights, material type, purpose for every vehicle |
| **Gate Pass Automation** | Hydra weights per size + manual weighbridge + auto-adjustment calculation + print |
| **Security Alerts** | Immediate alert on >20kg discrepancy (one-off OR trend) or stock-out mismatch |
| **Edit Tracking** | All edits logged with timestamp + user, visible to Owner |

**Gate Pass Workflow (Detailed):**

```
1. Office Staff selects order to load
2. Hydra Weight Guy sees order on dedicated mobile app → enters weight per size as he loads
3. Weights auto-sync to Office Staff's screen
4. Office Staff reviews weights
5. Weighbridge reading entered (manual in MVP-Core)
6. System compares: Hydra total vs Weighbridge
   - If >20kg variance → 🔴 Immediate Alert to Owner + Father
7. Rates auto-populated per size
8. Gate Pass auto-generated → Office Staff prints
9. (Optional) Third-party weighbridge entered → compared to internal weighbridge
   - Third-party is total only (multiple deliveries combined)
10. All edits tracked with timestamp + user
```

**Multi-Delivery Handling:**
- One order can have multiple deliveries
- Each delivery weighed separately on internal weighbridge after Hydra
- Third-party weighing (if requested) is total only — compared to sum of internal weighbridge readings

### Technical Approach

**Build vs Buy Decision:**

| Option | Consideration |
|--------|---------------|
| **Open Source ERP (ERPNext, Odoo, etc.)** | Research GitHub options; may fight custom CV/weighbridge integrations |
| **Next.js Custom Build** | More control for custom integrations; use proven UI libraries (Shadcn/ui) |
| **Backend** | Supabase or PostgreSQL — simple, scalable |
| **Hydra Mobile App** | Single-purpose dedicated app — React Native or PWA |

**Recommendation:** Research open-source ERPs on GitHub for reusable CRUD patterns, but lean toward Next.js custom build with component libraries for flexibility on CV and weighbridge integrations. Don't reinvent CRUD, but don't let ERP limitations block custom features.

**GitHub ERPs to Research:**
- ERPNext (Python/Frappe) — comprehensive but heavy
- Odoo Community — modular, learning curve
- Crater, InvoiceNinja, Akaunting — focused tools, may inspire patterns
